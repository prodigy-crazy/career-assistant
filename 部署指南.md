## 一、Render部署（推荐，免费方案）

### 步骤1：注册Render账号
1. 访问 https://render.com
2. 点击 "Sign Up" 使用GitHub账号登录

### 步骤2：部署后端服务
1. 登录后点击 "New +" → "Web Service"
2. 连接GitHub仓库：选择 `career-assistant`
3. 配置如下：
   - **Name**：`career-assistant-backend`
   - **Root Directory**：`backend`
   - **Runtime**：`Node`
   - **Build Command**：`npm install && node init-db.js`
   - **Start Command**：`npm start`
   - **Plan**：`Free`
4. 点击 "Create Web Service"

### 步骤3：部署前端静态页面
1. 点击 "New +" → "Static Site"
2. 选择同一个GitHub仓库
3. 配置如下：
   - **Name**：`career-assistant-frontend`
   - **Root Directory**：`./`（留空或填`.`）
   - **Build Command**：留空
   - **Publish Directory**：`.`
   - **Plan**：`Free`
4. 点击 "Create Static Site"

### 部署完成后获取的链接

| 服务 | 预期链接格式 |
|------|-------------|
| 前端页面 | `https://career-assistant-frontend.onrender.com` |
| 后端API | `https://career-assistant-backend.onrender.com` |

## 二、Vercel部署（备选方案）

### 后端部署
1. 访问 https://vercel.com
2. 使用GitHub账号登录
3. 点击 "Add New..." → "Project"
4. 选择 `career-assistant` 仓库
5. **Root Directory** 设置为 `backend`
6. 点击 "Deploy"

### 前端部署
1. 点击 "Add New..." → "Project"
2. 选择同一仓库
3. **Root Directory** 设置为根目录
4. Framework Preset 选择 "Other"
5. 点击 "Deploy"

## 八、常见问题

### Q1：SQLite数据持久化问题
Render免费版使用临时文件系统，数据库数据会在重启后丢失。解决方案：
- 使用Render的持久化磁盘（付费）
- 或使用外部数据库（如PostgreSQL、MongoDB Atlas免费版）

### Q2：前端无法连接后端API
确保前端API地址配置正确：
- 本地开发：`http://localhost:3001/api`
- 云端部署：`/api`（相对路径，通过反向代理）

当前代码已自动适配环境，无需手动修改。

### Q3：验证码功能说明
当前验证码为模拟功能，仅用于演示。生产环境需接入真实短信服务。
